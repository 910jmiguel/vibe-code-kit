# PRD Phase

You're helping me write a Product Requirements Document. Before we dive in, I need you to understand what we're building.

**Answer these questions first:**

1. What's the project? One-liner description.
2. What's the tech stack? (Language, framework, database, hosting)
3. Do you have an existing plan or notes? If so, paste the key points here. If not, just describe what you want to build.

**Wait for my answers before continuing.**

---

Once I've answered, generate a complete PRD with the following sections:

## PRD Structure

### 1. Overview
- **Project Name:**
- **One-liner:** What it does in one sentence
- **Target Users:** Who uses this and why
- **Success Criteria:** How do we know it's working? (measurable where possible)

### 2. Problem Statement
What problem does this solve? Why does it matter? What's the current alternative (even if it's "nothing" or "a spreadsheet")?

### 3. Features & Requirements

For each feature:
- **Feature Name**
- **User Story:** As a [user], I want to [action] so that [benefit]
- **Requirements:** Specific, testable requirements (numbered)
- **Acceptance Criteria:** How do we verify this works?
- **Priority:** Must-have / Should-have / Nice-to-have
- **Notes:** Edge cases, constraints, open questions

Group features by area (e.g., "Authentication", "Dashboard", "API") if it makes sense.

### 4. Technical Requirements
- Performance targets (if any)
- Security requirements
- Compatibility / browser support
- Third-party integrations
- Data storage and privacy considerations

### 5. UI/UX Requirements
- Key screens or views (describe, don't design)
- Navigation flow
- Responsive requirements
- Accessibility considerations

### 6. Out of Scope
What are we explicitly NOT building in this version? Be specific so scope doesn't creep.

### 7. Open Questions
Things we still need to decide. Tag each with who needs to answer it if possible.

### 8. Timeline & Phases
Map features to implementation phases. Reference the plan if one exists.

---

**After generating the PRD:**
- Ask me to review each section
- Flag anything you had to assume or guess
- Suggest what to do next (usually: create a checklist with `/checklist` or the checklist prompt)
